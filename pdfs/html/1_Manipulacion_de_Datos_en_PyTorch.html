El Aprendizaje Profundo se engloba dentro del Aprendizaje Computacional. Tiene como objetivo generar un modelo para resolver un problema a partir de datos. Muchos datos. Inmensas cantidades de datos!

Por eso es vital cmo manejar y almacenar esos datos.

En PyTorch y otras libreras similares los clculos se hacen sobre datos en punto flotante (nmeros reales) y la mayora de los procesos implican operar arrays n-dimensionales. Por convencin nos referimos a esta estructura de datos como tensor. Pese al nombre especfico, un tensor no es ms que un array multidimensional. Un vector es un array de 1 dimensin, una matriz es un array de 2 dimensiones y un tensor es un array de cualquier nmero de dimensiones.

Aunque su funcionamiento interno sea distinto, la usabilidad de los tensores en PyTorch es similar a la de los arrays de Numpy. Se puede obtener un tensor de PyTorch a partir de un array de Numpy y viceversa.

Entonces por qu no usamos Numpy en lugar de los tensores de PyTorch? Porque los tensores de PyTorch pueden mandarse a la GPU para realizar las operaciones en ella. Podemos consultar dnde est alojado el tensor y moverlo de memoria principal a memoria de video.

In[8]:
print(f"torch_tensor est alojado en: {torch_tensor.device}")
torch_tensor_gpu = torch_tensor.to("cuda")
print(f"torch_tensor est ahora alojado en: {torch_tensor_gpu.device}")

torch_tensor est alojado en: cpu
torch_tensor est ahora alojado en: cuda:0

Ntese que la sentencia para mover a la GPU el tensor implica una asignacin. La operacin no se hace sobre el mismo objeto tensor, sino que se genera uno nuevo. Esto tiene sentido si uno lo piensa. El tensor es un objeto en memoria, pero no queremos modificar ese objeto, sino "moverlo" a otra memoria. Estamos generando un nuevo tensor con la misma informacin.

Es llamativo tambin que para indicar que el tensor est alojado en memoria de video se usa el indicativo 'cuda:0'. 'cuda' indica que est en memoria de video y el 0 indica el ndice del dispositivo en el que est. Si dispusieramos de varias GPUs podramos mover tensores entre distintas GPUs.

Ya que PyTorch permite operar en la GPU y nos interesa para acelerar nuestros clculos, vamos a probar cmo evolucionan los tiempos de operacin con distintos tamaos de matrices en Numpy, PyTorch en CPU y PyTorch en GPU.

In[10]:
SIZES = [i*500 for i in range(30)]

numpy_times = []      # Lista para almacenar los tiempos de Numpy.
torch_cpu_times = []  # Lista para almacenar los tiempos de PyTorch en CPU.
torch_gpu_times = []  # Lista para almacenar los tiempos de PyTorch en GPU.

for size in SIZES:

  # Operamos en Numpy
  A_np = np.random.rand(size, size).astype(np.float32)
  B_np = np.random.rand(size, size).astype(np.float32)

  start_time = time.time()
  C_np = A_np*B_np
  end_time = time.time()
  numpy_times.append(end_time-start_time)

  # Operamos en PyTorch en CPU
  A_torch_cpu = torch.from_numpy(A_np)
  B_torch_cpu = torch.from_numpy(B_np)

  start_time = time.time()
  C_torch_cpu = A_torch_cpu*B_torch_cpu
  end_time = time.time()
  torch_cpu_times.append(end_time-start_time)

  # Operamos en PyTorch en CPU
  A_torch_gpu = A_torch_cpu.to('cuda')
  B_torch_gpu = B_torch_cpu.to('cuda')

  start_time = time.time()
  C_torch_cpu = A_torch_gpu*B_torch_gpu
  end_time = time.time()
  torch_gpu_times.append(end_time-start_time)

# Grfica de resultados
plt.figure(figsize=(8,6))
plt.plot(SIZES, numpy_times, marker='o', label="NumPy (CPU)")
plt.plot(SIZES, torch_cpu_times, marker='s', label="PyTorch (CPU)")
plt.plot(SIZES, torch_gpu_times, marker='^', label="PyTorch (GPU)")
plt.xlabel("Tamao de la matriz (N x N)")
plt.ylabel("Tiempo (segundos)")
plt.title("Comparacin de tiempos: NumPy vs PyTorch (CPU vs GPU)")
plt.legend()
plt.grid(True)
plt.show()

Como se puede observar, el tiempo de ejecucin tiene un aumento mucho menor al trabajar con PyTorch en GPU respecto a Numpy y PyTorch en CPU. Es interesante observar que los tiempos de PyTorch en CPU son peores que los de Numpy aunque ambos funcionan en CPU. Pytorch, en su versin de CPU, es menos eficiente que Numpy.

Sin embargo, esta comparacin no es del todo justa. Con ella en mente lo lgico sera usar siempre que sea posible Pytorch en GPU.

SIZES = [i*500 for i in range(30)]
N_OPS = 50
numpy_times = []      # Lista para almacenar los tiempos de Numpy.
torch_cpu_times = []  # Lista para almacenar los tiempos de PyTorch en CPU.
torch_gpu_times = []  # Lista para almacenar los tiempos de PyTorch en GPU.

for size in SIZES:

  # Operamos en Numpy
  A_np = np.random.rand(size, size).astype(np.float32)
  B_np = np.random.rand(size, size).astype(np.float32)

  start_time = time.time()
  for i in range(N_OPS):
    C_np = A_np*B_np*i
  end_time = time.time()
  numpy_times.append(end_time-start_time)

  # Operamos en PyTorch en CPU
  A_torch_cpu = torch.from_numpy(A_np)
  B_torch_cpu = torch.from_numpy(B_np)

  start_time = time.time()
  for i in range(N_OPS):
    C_torch_cpu = A_torch_cpu*B_torch_cpu*i
  end_time = time.time()
  torch_cpu_times.append(end_time-start_time)

  # Operamos en PyTorch en GPU
  start_time = time.time()
  for i in range(N_OPS):
    A_torch_gpu = A_torch_cpu.to('cuda')
    B_torch_gpu = B_torch_cpu.to('cuda')

    C_torch_cpu = A_torch_gpu*B_torch_gpu*i
  end_time = time.time()
  torch_gpu_times.append(end_time-start_time)

# Grfica de resultados
plt.figure(figsize=(8,6))

plt.plot(SIZES, torch_cpu_times, marker='s', label="PyTorch (CPU)")

plt.xlabel("Tamao de la matriz (N x N)")

plt.title("Comparacin de tiempos: NumPy vs PyTorch (CPU vs GPU)")
