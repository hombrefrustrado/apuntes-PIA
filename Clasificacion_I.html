Hasta ahora todo lo que hemos abordado han sido problemas supervisados (conocemos la entrada y la salida) y de regresin. Los problemas de regresin consisten en modelar la relacin entre distintas variables dependientes para poder predecir una de ellas en funcin a las otras. Aprender funciones en el sentido ms directo y estricto del trmino.

Sin embargo, no son los nicos problemas que se abordan en Aprendizaje Computacional. Un tipo de problema tpico del Aprendizaje Computacional y del Aprendizaje Profundo son los problemas de clasificacin.

Un problema de clasificacin consiste en, dado un ejemplo (una serie de valores, una imagen, un video, etc) clasificarlo en una categora. Asignarle una etiqueta habitualmente de entre un conjunto predefinido de opciones. Inferir si una persona est enferma o no a partir de su historial mdico es un problema de clasificacin en el que las categoras son sano y enfermo. Concretamente un problema de clasificacin binario.

Los problemas de clasificacin se dividen entre los binarios (solo dos opciones) y los multiclase (ms de dos categoras). La divisin puede parecer arbitraria, pero las herramientas necesarias para abordar uno u otro cambian. Vamos a comenzar viendo cmo afrontar un problema de clasificacin usando Aprendizaje Profundo en PyTorch.

Para empezar necesitamos un conjunto de datos. Crear un conjunto de datos artificiales para un problema de clasificacin es sensiblemente ms complicado que crearlo para los ejemplos de regresin, as que vamos a recurrir a los conjuntos de datos que PyTorch provee :)

def filter_dataset(dataset, chosen_labels):
  """
  Devuelve un subconjunto del dataset que solo contiene las etiquetas especificadas.
  """
  indices = [i for i, (_, y) in enumerate(dataset) if y in chosen_labels]
  subset = torch.utils.data.Subset(dataset, indices)
  return subset

def split_dataset(dataset, split_share=0.5):
  """
  Devuelve dos subconjuntos del dataset. split_share define cuntos ejemplos irn al
  primer subconjunto. El resto irn al segundo.
  """
  mask_indices_to_first_subset = torch.rand(len(dataset))<=split_share
  indices_first_subset = [i for i, (_, _) in enumerate(dataset) if mask_indices_to_first_subset[i]]
  indices_second_subset = [i for i, (_, _) in enumerate(dataset) if not mask_indices_to_first_subset[i]]

  first_subset = torch.utils.data.Subset(dataset, indices_first_subset)
  second_subset = torch.utils.data.Subset(dataset, indices_second_subset)

  return first_subset, second_subset

def learning_loop_for_binary_classification(train_dataloader, val_dataloader, model, epochs, loss_fn, learning_rate, optimizer, positive_class):
  epoch_loss_list = []
  val_loss_list = []
  val_acc_list = []

  opt = optimizer(
    model.parameters(),
    lr=learning_rate
  )

  with tqdm(range(epochs), desc="epoch:") as pbar:
    for epoch in pbar:
      steps_loss_list = []
      for x_true, y_true in train_dataloader:
        y_true = y_true[:,None]                                     # Pequeo arreglo que necesitamos porque el MNIST no est pensado para clasificacin binaria.
        y_true = 1*(y_true == positive_class).to(torch.float32)     # Convertimos la clase positiva en 1 y la otra en 0.
        y_pred = model(x_true)                      # Forward
        opt.zero_grad()
        loss = loss_fn(y_pred, y_true)
        loss.backward()                                     # Backward
        steps_loss_list.append(loss.clone().detach())
        opt.step()
      train_loss = torch.tensor(steps_loss_list).mean()
      epoch_loss_list.append(train_loss)
      pbar.set_postfix(loss=f"{train_loss}")

      if epoch%5==0:
        val_step_loss_list = []
        val_correct_answers = 0
        val_set_size = 0
        with torch.no_grad():
          for x_val_true, y_val_true in val_dataloader:
            y_val_true = y_val_true[:,None]                                         # Pequeo arreglo que necesitamos porque el MNIST no est pensado para clasificacin binaria.
            y_val_true = 1*(y_val_true == positive_class).to(torch.float32)         # Convertimos la clase positiva en 1 y la otra en 0.
            y_val_pred = model(x_val_true)

            loss_val = loss_fn(y_val_pred, y_val_true)
            val_step_loss_list.append(loss_val)

            TP = torch.sum((y_val_true>=0.5) & (y_val_pred>=0.5))
            TF = torch.sum((y_val_true<0.5) & (y_val_pred<0.5))
            correct_answers = TP+TF
            val_correct_answers += correct_answers
            val_set_size += y_val_true.numel()

          val_loss_list.append(torch.tensor(val_step_loss_list).mean())
          val_acc_list.append(val_correct_answers/float(val_set_size))

  return model, epoch_loss_list, val_loss_list, val_acc_list

S, est un poco cambiado, pero bsicamente hemos aadido tres cosas:

  • Incluimos un argumento para determinar cul es la clase positiva (los 1s).
  • Con el conjunto de validacin adems de la prdida almacenemos el accuracy (el nmero de aciertos).
  • La barra de progreso ir mostrando el valor de error que vamos obteniendo. No es lo ms eficiente, pero es til para tener una idea rpida de si el entrenamiento va progresando.

La idea es sencilla. En un problema de calsificacin podemos tener una nocin intuitiva de acierto. Si hay dos opciones y nuestro modelo elige la correcta, acierta. Cules son las opciones? Como hemos dicho antes, tenemos que trabajar con nmeros as que 0 y 1 son los valores elegidos por convencin. 1 se conoce como la clase positiva y 0 la clase positiva. Nuestro modelo es muy dificil que tenga como resultado un 1 absoluto, as que asumimos que si su respuesta est ms cerca del 1 que del 0 (mayor que 0.5).

Comparando los resultados con las etiquetas, ahora podemos recurrir a mtricas de clasificacin:

  • True Positives (TP): Nmero de ejemplos de la clase positiva clasificados como tal.
  • True Negatives (TN): Nmero de ejemplos de la clase negativa clasificados como tal.
  • False Positives (FP): Nmero de ejemplos de la clase negativa clasificados como positivos.
  • False Negatives (FN): Nmero de ejemplos de la clase positiva clasificados como negativos.

$$Accuracy = \frac{TP+TN}{TP+TN+FP+FN}$$

$$Recall = \frac{TP}{TP+FN}$$

$$Precision = \frac{TP}{TP+FP}$$

Estas mtricas son recomendables para entender el comportamiento del modelo que hemos entrenado para la clasificacin binaria, pero no se usan para guiar el entrenamiento en Aprendizaje Profundo porque no tiene sentido considerar ninguna de ellas para un nico ejemplo, sino para un conjunto de ejemplos. El clculo del gradiente durante el paso hacia atrs usando dichas funciones da poca informacin de cara a ajustar los parmetros del modelo.

Pero bueno, las funciones de prdida basadas en la distancia como el error absoluto y el error cuadrtico medio siguen teniendo sentido. Simplemente nuestros valores predichos ahroa estarn entre 0 y 1 y las etiquetas objetivo sern 0 o 1.

Podemos dar por hecho que nuestros valores estarn entre 0 y 1 porque disearemos un modelo neuronal con dicho comportamiento. Cmo? Simplemente eligiendo una funcin de activacin adecuada para la ltima capa. La sigmoide, por ejemplo, est tiene resultados justamente en el rango [0,1]. Tambin podramos usar la tangente hiperblica y reescalar la salida para convertir el rango [-1,1] a [0,1].

Vamos a hacer un ejemplo en el que asignamos la clase positiva (1) a los ejemplos con etiqueta 7 y la clase negativa (0) a los ejemplos con etiqueta 1. Esta asignacin es completamente arbitraria. Si fueran imgenes de perros y gatos podramos asignar cualquiera de los dos animales como clase positiva.

Podemos ver que el accuracy del conjunto de validacin aumenta a un valor muy cercano al 1 mientras el error disminuye. Sin embargo, el error no llega a bajar de 0.4. Nuestro modelo funciona bastante bien. Acierta ms del 95% de las veces, pero nuestra funcin de error no lo refleja. Por qu? Bueno, aceptamos como clasificacin positiva cualquier valor que supere el 0.5 (tcnicamente tambin el 0.5) y como clasificacin negativa cualquier valor menor que 0.5. Un resultado de 0.51 es tan bueno para clasificar un ejemplo de la clase positiva como un 0.99 (y as lo evala el accuracy), pero para el error absoluto es un error de 0.49. Para ese mismo ejemplo un resultado de 0.48 es un error de falso negativo, pero para el error absoluto es un error de 0.52. Una diferencia crucial en la clasificacin para el error absoluto no supone prcticamente una diferencia.

Aunque un error basado en la distancia funcione en un problema tan sencillo como el que estamos trabajando, no es especialmente bueno caracterizando las propiedades de un problema de clasificacin.

Por eso vamos a plantear una funcin de error que caracteriza mejor un problema binario como la entropia cruzada binaria (Binary Cross Entropy o BCE a partir de ahora).

$$BCE(y_{pred}, y_{true}) = -(y_{true}log(y_{pred})+(1-y_{true})log(1-y_{pred}))$$

Vamos a ver el comportamiento de esta funcin. Si suponemos un caso en el que la clasificacin es perfecta ($y_{pred} = y_{true}$), si la clasificacin es positiva ($y_{pred} = y_{true}=1$), el primer trmino de la ecuacin es $log(1)=0$ mientras que el segundo trmino es $0log(0)=(0)(-)$. Si la clasificacin es negativa ($y_{pred} = y_{true}=0$) los resultados de los trminos se intercambian. Con una clasificacin diametralmente incorrecta ($y_{pred} = 1$ con $y_{true}=0$ o $y_{pred} = 0$ con $y_{true}=1$) uno de los trminos es $-\infty$. En todos estos casos aparece un valor de $\infty$ debido a la asntota de la funcin logartimca. Esto es un problema desde el punto de vista del anlisis matemtico... pero para nuestros propsitos no lo es tanto. En la prctica PyTorch limita (funcin clamp en Pytorch y clip en TensorFlow) el valor de ese logaritmo para que no de valores menores de -100. Si cambiamos los $-\infty$ por -100 podemos observar que cuando la clasificacin es perfecta, la funcin BCE tiene como resultado 0, mientras que cuando la calsificacin es totalmente incorrecta tiene como resultado 100.

Vamos a observar la evolucin del BCE en la prctica comparado con el error cuadrtico y el absoluto segn se aleja de la clasificacin ideal.

El error absoluto crece de manera constante. Conceptualmente penaliza igual una diferencia entre 0.2 y 0.4 que una diferencia entre 0.4 y 0.6 (su pendiente es constante). Sin embargo, para el problema que estamos trabajando sabemos que la diferencia entre 0.2 y 0.4 nos importa mucho menos que la dieferencia entre 0.4 y 0.6. El error cuadrtico s que aumenta la penalizacin del error cunto mayor es (la inclinacin incrementa), aunque entre 0 y 1 el aumento no es especialmente grande. La BCE muestra un incremento de la curva an mayor.

Vamos a aprovechar para reflexionar sobre cmo afecta la funcin de error al aprendizaje. Como hemos dicho previamente, nuestro ajuste se basa en el gradiente del error respecto a los parmetros ajustables del modelo que produce ese error. Calcular el gradiente durante el paso hacia atrs (backward) implica aplicar la regla de la cadena y el gradiente respecto a los parmetros va a tener siempre la forma:

$$\nabla L(m_{\mathbf{w}}(x))= (\frac{L(m_{\mathbf{w}}(x))}{w_{1}},...,\frac{L(m_{\mathbf{w}}(x))}{w_{n}}) = (\frac{L(m_{\mathbf{w}}(x))}{m_{\mathbf{w}}(x)}\frac{m_{\mathbf{w}}(x)}{w_{1}},...,\frac{L(m_{\mathbf{w}}(x))}{m_{\mathbf{w}}(x)}\frac{m_{\mathbf{w}}(x)}{w_{n}})$$

dado un modelo $m_{\mathbf{w}}$ con los parmetros ajustables $\mathbf{w}=(w_{1},...,w_{n}))$. Todas las componentes del vector gradiente estn multiplicados por $\frac{L(m_{\mathbf{w}}(x))}{m_{\mathbf{w}}(x)}$ que es la derivada de la funcin de error respecto al resultado del modelo.

Con lo cul es relevante cmo evoluciona la derivada de la funcin de error segn el error de clasificacin aumenta. Vamos a hacer una pequea chapuza (porque PyTorch no est pensado para esto) para obtener el gradiente nicamente de la funcin de error segn este aumenta desde una prediccin correcta ($y_{pred} = 0$ con $y_{true}=0$) a una totalmente incorrecta ($y_{pred} = 1$ con $y_{true}=0$).

In[]:
bce = torch.nn.BCELoss()
l1 = torch.nn.L1Loss()
se = torch.nn.MSELoss()

bce_grad_list = []
l1_grad_list = []
se_grad_list = []

x = torch.linspace(0.001,0.95,100)
for fake_output in x:
  fake_output.requires_grad_(True)
  target = torch.zeros_like(fake_output)

  bce_error = bce(fake_output, target)
  bce_error.backward()
  bce_grad = fake_output.grad.clone()
  bce_grad_list.append(bce_grad)
  fake_output.grad.zero_()

  l1_error = l1(fake_output, target)
  l1_error.backward()
  l1_grad = fake_output.grad.clone()
  l1_grad_list.append(l1_grad)
  fake_output.grad.zero_()

  se_error = se(fake_output, target)
  se_error.backward()
  se_grad = fake_output.grad.clone()
  se_grad_list.append(se_grad)
  fake_output.grad.zero_()

x = x.detach()
plt.figure(figsize=(8,6))
plt.plot(x, bce_grad_list, label="BCE")
plt.plot(x, l1_grad_list, label="L1")
plt.plot(x, se_grad_list, label="SE")
plt.legend()
plt.title("Evolucin del Gradiente")
plt.grid(True)
plt.show()

Nota: una vez ms restringimos el rango para favorecer la visibilidad de la grfica.

El gradiente del error absoluto es una constante. Es lo esperable en una funcin que para casos positivos es una recta de inclinacin 1. Su derivada es la constante 1. Esto nos indica que si la clasificacin correcta es 0, da igual si nuestro modelo indica 0.2, 0.7 o 1, el factor que multiplicar las derivadas paricales del modelo ser $\frac{L(m_{\mathbf{w}}(x))}{m_{\mathbf{w}}(x)}=1$ para todos los casos.

El gradiente del error cuadrtico es ahora una recta de inclinacin 2. Consecuencia de que $x^2 dx = 2x$. El factor que multiplicar las derivadas parciales del modelo no ser el mismo si nuestro modelo indica 0.2 (menor que 1), 0.5 (exactamente 1) o 1 (exactamente 2). Dado un lote de 2 ejemplos ($x_{1}$ y $x_{2}$) en los que el modelo $m_{\mathbf{w}}$ tiene resultados 0.5 y 1 respectivamente. Si la clasificacin correcta es 0, la derivada del error en esos puntos ser $\frac{L(m_{\mathbf{w}}(x_{1}))}{m_{\mathbf{w}}(x_{1})}=1$ y $\frac{L(m_{\mathbf{w}}(x_{2}))}{m_{\mathbf{w}}(x_{2})}=2$. Eso hara que las derivadas parciales del modelo en el punto $x_{2}$ para el gradiente tengan el "doble de influencia" en el gradiente final. Esto tiene sentido si queremos priorizar la direccin de ajuste que indica el gradiente de los ejemplos con los que nuestro modelo funciona peor.

Est lgica nos interesa especialmente en la clasificacin binaria. Que nuestro modelo dictamine 0.2 cuando debera decir 0 sigue siendo acertar, por lo que corregir en base a su gradiente nos interesa menos que corregir en base a los ejemplos en los que dictamina mayor que 0.5.

Observando la evolucin del gradiente de la BCE, sabemos que se dar mucha ms importancia a los ejemplos con clasificaciones muy lejanas a la ideal.
